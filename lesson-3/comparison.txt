# Сравнение Docker-образов для PyTorch инференса

1. Размер образов:
   - Fat-образ (Dockerfile.fat): 8.53 GB
   - Slim-образ (Dockerfile.slim): 1.1 GB

2. Анализ слоев:
   - Fat-образ: использует базовый образ python:3.11 (Debian), включает в себя полные библиотеки CUDA/NVIDIA для GPU, которые весят несколько гигабайт, даже если не используются.
   - Slim-образ: использует python:3.11-slim, multi-stage build и установку только CPU-версии PyTorch через --index-url.

3. Лишние инструменты:
   - В Fat-образе остались кэши pip и полные версии библиотек для разработки.
   - В Slim-образе отсутствуют инструменты компиляции и кэши, оставлен только рантайм.

4. Предложения по оптимизации:
   - Использование более компактных базовых образов (например, distroless).
   - Квантование модели (Quantization) для уменьшения размера самого файла model.pt.
   - Удаление ненужных файлов внутри образа через .dockerignore.
